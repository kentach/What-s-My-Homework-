## 1. サービス概要（※最後に書いてOK）
本アプリは、前職の英会話スクールで感じた「宿題管理の非効率さ」を解決するために開発した、前職の英会話スクール専用の宿題管理アプリです。生徒はいつでも自分の宿題や単語テスト範囲を確認でき、毎週取り組むべき学習内容を自身で管理できます。また、欠席時でも校舎へ問い合わせることなく必要な情報を確認できるため、学習機会の損失を防ぐことができます。また講師側は宿題やテスト範囲をオンライン上で一元管理できるため、印刷業務や問い合わせ対応の負担を軽減できます。本アプリは、生徒・保護者・講師の三者がより効率的に学習・指導を行える環境の実現を目指して開発しました。

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情
本アプリを開発しようと思ったきっかけは、前職の英会話スクールでの勤務経験です。当時、生徒への宿題管理や単語テストの運用を行う中で、紙ベースの管理に多くの課題があることを実感しました。
1. 宿題内容の修正対応が煩雑
宿題プリントは事前にダブルチェックを行ったうえで配布していましたが、その後にカリキュラム作成者から内容の修正連絡が入ることが頻繁にありました。その場合、講師は授業内で口頭説明を行うか、修正版を再印刷して生徒へ配布しなければなりません。修正のたびに追加対応が発生し、講師・生徒双方に負担がかかっていました。
2. 印刷コストと管理コストが発生する
宿題プリントは毎月配信されるカリキュラムに合わせて作成・印刷されます。現場では担当講師が印刷を担当していましたが、印刷枚数が不足すると再度印刷しなければならないため、多めに印刷するケースが少なくありませんでした。また、全16校舎で毎月宿題プリントを印刷していることを考えると、用紙代やインク代などの印刷コストも無視できない課題でした。
3. 欠席した生徒が宿題内容を確認しづらい
生徒が授業を欠席すると、次回までに取り組むべき宿題が分からなくなります。その際、生徒や保護者は校舎へ電話をかけて確認する必要があり、スタッフはクラスファイルを確認して口頭で伝えたり、連絡アプリを通じて宿題内容を送信したりしていました。このような問い合わせは頻繁に発生しており、授業準備や他業務を行っている講師にとって大きな負担となっていました。また、生徒の中には「休んだから宿題はやらなくてもいい」と考え、宿題を行わずに次回の授業へ参加するケースも見られました。
4. 月例単語テストの運用にも課題があった
前職の英会話スクールでは、毎月末に単語テストを実施していました。テスト範囲は担当講師が範囲表を印刷して配布するか、授業内でノートに記入させる形で伝えていました。しかし、宿題管理と同様に、
- テスト範囲の修正が発生する
- 講師が範囲を伝え忘れる
- 生徒が欠席して範囲を把握できない
といった問題が頻繁に起こっていました。その結果、生徒がテスト当日に初めて範囲を知るケースもあり、十分な準備ができないまま受験することもありました。
このように、宿題や単語テストの管理を紙ベースで行う運用には、
- 情報修正時の対応負担
- 印刷コスト
- 欠席者への情報共有
- 講師側の管理工数
といった課題がありました。
そこで私は、「生徒がいつでも最新の宿題やテスト範囲を確認でき、講師側も一元管理できる仕組みがあれば、双方の負担を大きく軽減できるのではないか」と考え、本アプリの開発を決意しました。

  3. 宿題の一つとして、自宅で音読学習を取り入れるように促す。

1. 現場の講師が必要としているかどうか。このアプリを導入することでどんな価値があるのかを知ってもらう。

### 10-3.キャッチアップ不足の懸念

キャッチアップ不足の段階では以下の懸念点があります。
1. **完成までのハードルが高い**
使用する言語やフレームワークが異なるため、キャッチアップにかかる時間は個人差があるものの、一定の時間を要することが予想されます。
2. **Issueの見積もりが難しい**
「どこまで理解していて、何が分からないのか」「調査にかかる時間」などが不明瞭な場合、開発期間内に完了することが難しくなってしまいます。

例えば、最近注目されているReactやNextについて。
Reactについては**本リリース後に組み込む方向で検討**するのが良いかと考えています。
その後、React単体のキャッチアップを行い、アプリに組み込む技術的な検証をした上で、組み込むかどうかを判断する形が理想かもしれません。
